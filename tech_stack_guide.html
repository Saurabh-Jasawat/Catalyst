1. System Architecture & End-to-End Handshake

The Catalyst Study Tracker is engineered as a decoupled single-page application built on the classic MERN (MongoDB, Express, React, Node.js) stack. The frontend compiles into modular, optimized bundles via Vite, communicating with the headless Express API on port 5000 using asynchronous network requests.

6. Core Technical Concept Explanations & Interview Prep

Q: Why is BcryptJS used over simple cryptographic libraries like crypto?

BcryptJS implements a computational cost factor (salt rounds) that scales key derivation complexity over time. Unlike standard MD5 or SHA256 hashes, bcrypt uses an iterative hashing algorithm. This slows down brute force attacks significantly. The computation cost factor ensures that even if hackers export the raw MongoDB user schemas, decrypting password hashes remains computationally expensive.

Q: What is the benefit of Tailwind CSS v4.0 in MERN builds?

Tailwind CSS v4.0 introduces an optimized PostCSS compiler engine. Instead of scanning files for selectors at runtime, v4 resolves utility structures at build time. This compiles the stylesheet into a highly optimized production bundle. It provides support for custom CSS variables, layout grids, container queries, and native dark-theme media configurations directly inside Tailwind class strings.

Q: How does CORS work, and what happens if it is misconfigured?

Cross-Origin Resource Sharing (CORS) is a browser security protocol that prevents malicious scripts on one origin from accessing resources on another. If the React frontend on `localhost:5174` requests data from the Express backend on `localhost:5000` without CORS configured on the backend, the browser will block the response. This displays a cross-origin resource error in the console, preventing data from loading into the user interface.

Q: Why is local state used alongside database synchronization?

React state provides optimistic UI updates. When a student checks a study planner milestone, the UI updates immediately to a checked state (optimistic render) while the backend updates in the background. If the API request succeeds, the local state remains checked. If it fails, the local state rolls back, prompting an error message. This keeps the application responsive and interactive.
