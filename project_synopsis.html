Project Synopsis

Catalyst: Smart Syllabus Tracking & Exam Prep Analyzer

MERN Stack Web Application

1. Project Abstract

Preparation for high-stakes competitive examinations requires a high degree of consistency, structured revision, and self-evaluation. However, students often struggle to maintain visibility over their progress across massive syllabi. Catalyst resolves this challenge by providing a centralized web-based tracker built on the MERN stack. Catalyst dynamically monitors syllabus coverage, structures spaced repetition cycles, organizes daily study tasks, and logs previous year question (PYQ) performance.

2. Objectives of the Project

  • To provide an interactive syllabus tracking engine where students can visualize their progress down to subtopic-level details.
  • To design a Spaced Repetition Scheduler that dynamically flags topics due for revision to improve retention.
  • To implement a MERN-based synchronization pipeline that automatically saves user state context (notes, scores, streaks) to MongoDB.
  • To create an OTP-based password recovery protocol allowing users to reset credentials securely via Gmail.

3. Scope of the System

The system handles user authentication, multi-exam workspace profiles, syllabus mapping, mock test metrics logs, error/trap registers, and automated backups. The scope excludes online test hosting (questions database) and focuses on progress tracking, analytical reports, and study scheduling.

4. Technical Stack Requirements

Software Requirements:

  • Operating System: Windows, Linux, or macOS
  • Database Engine: MongoDB (version 6.0+)
  • Backend Runtime: Node.js (version 18+) & Express
  • Frontend Library: React (Vite-powered)
  • Styling: Vanilla CSS

Hardware Requirements:

  • Processor: Dual-Core CPU (2.0 GHz minimum)
  • System RAM: 4 GB minimum (8 GB recommended)
  • Disk Storage: 500 MB free space (for node modules and files)
  • Internet: Port 5000 and 5174 local network access

5. Project Methodology

We followed the Agile software development lifecycle, utilizing component-driven architecture for the React client and RESTful API routers for Node/Express. Mongoose schemas model data transactions cleanly. All updates are synced via a debounced context scheduler to maintain network efficiency.

6. Expected Outcomes

Users will benefit from a unified, zero-crash dashboard reflecting their preparation status, dynamic streaks and XP scores, structured daily task pipelines, organized exam trackers, and a fully working secure recovery flow.
