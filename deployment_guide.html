1. Production Monorepo Deployment Architecture

For a local development environment, the React frontend and Express backend run on separate local servers (ports 5174 and 5000). For live cloud deployments, hosting them on separate servers increases setup complexity and communication overhead.

In our production setup, we compile the React frontend into static HTML, CSS, and JS assets using Vite (`npm run build`). Express then serves these static assets from the `frontend/dist` folder when running in production mode. This maps both API calls and user page navigations to a single, unified web service.

2. Deployment Steps

  • 1
    Initialize Git and Commit Workspace Files

    Render monitors your Git repositories for updates, triggering builds automatically when you push changes. We need to initialize your repository locally and stage the workspace. A root `.gitignore` file has already been added to protect your configurations.

    PowerShell - d:\GLA\Miscelleneous\project\gambit
    PS > git init
    Initialized empty Git repository in d:/GLA/Miscelleneous/project/gambit/.git/
    PS > git add .
    PS > git commit -m "Configure workspace for cloud monorepo deployment"
    [main (root-commit) a1c2e3f] Configure workspace for cloud monorepo deployment
  • 2
    Publish to GitHub

    Create a new repository on your GitHub account. Make sure to **leave it empty** (do not add a README or `.gitignore` file, as the workspace already contains them). Pushing to this repository makes it accessible to Render.

    PowerShell - Publish to GitHub
    PS > git remote add origin https://github.com/<your-username>/<your-repo-name>.git
    PS > git branch -M main
    PS > git push -u origin main
    Counting objects: 1256, done. Writing objects: 100% (1256/1256), 2.45 MiB | 512.00 KiB/s, done. To https://github.com/<your-username>/<your-repo-name>.git * [new branch] main -> main
  • 3
    Provision MongoDB Atlas Database

    Since a local database runs on `localhost` (`mongodb://127.0.0.1:27017`), cloud-hosted applications require a live cluster to connect to. We will provision a database instance on MongoDB Atlas.

    Setting up a Database Cluster:

    • Register: Go to mongodb.com/cloud/atlas and register for a free account.
    • Create M0 Cluster: Click Create a Database and select the **M0 Shared Free Tier**. Choose your cloud provider (e.g., AWS) and region (e.g., North Virginia).
    • Credentials: In the security panel, create a database access user (e.g., `catalyst_admin` with password `SecureDbPass123`). Keep these credentials safe!
    • Network Firewall: Go to **Network Access** and add a new IP. Set it to `0.0.0.0/0` (Allow Access from Anywhere) to permit Render's servers to connect.
    • Get Connection String: Go to your Database view, click **Connect** -> **Drivers**, and copy the connection string:
      mongodb+srv://catalyst_admin:<password>@cluster0.xxxx.mongodb.net/catalyst?retryWrites=true&w=majority
  • 4
    Configure and Deploy Web Service on Render

    Render will install all frontend and backend dependencies, build the Vite application, and launch the Node.js server.

    • Register/Login: Go to render.com and sign in using your GitHub account.
    • New Service: Click New + at the top right, and select **Web Service**.
    • Link Repository: Search for and connect the repository you pushed to in Step 2.
    • Configure Settings: Configure the Web Service settings as follows:
    Setting Name Configuration Value Technical Purpose
    Name catalyst-tracker The name of your service (defines your live URL).
    Language Node Sets the server runtime to Node.js.
    Build Command npm run build Runs the root build script, building the React app to `dist`.
    Start Command npm start Starts the Express backend on the designated port.
    Instance Type Free Uses Render's free hosting tier.
  • 5
    Configure Environment Variables

    Environment variables are used to securely pass configuration details like database connection strings and secret keys to the application without hardcoding them in the source code. On the Render dashboard, go to the **Environment** tab and add the following keys:

    Environment Variable Name Value Example Purpose in Application
    NODE_ENV production Instructs Express to serve compiled static frontend bundles.
    MONGODB_URI mongodb+srv://catalyst_admin:SecureDbPass123@... Directs Express to connect to your live MongoDB Atlas cluster.
    JWT_SECRET super_secure_long_random_jwt_key_987654 Generates secure authentication tokens for active users.
    SMTP_USER aspirantsupport@gmail.com The Gmail address used to send verification and OTP codes.
    SMTP_PASS xxxx xxxx xxxx xxxx The 16-character Google App Password (not your personal password).
  • 6
    Launch and Monitor Logs

    Once you click **Deploy Web Service**, Render launches the build pipeline. You can monitor the process in the terminal log view.

3. Cloud Security, Whitelisting, and SMTP Passwords

⚠️ Critical SMTP / Gmail App Passwords:

Google accounts block standard login requests from unknown cloud servers. To send password reset and email OTP codes using Gmail SMTP, you must configure a **Google App Password**. Do not input your raw personal Gmail password into the `SMTP_PASS` variable, as it will be rejected.

  • Go to your Google Account Settings -> Security.
  • Enable 2-Step Verification.
  • Click App passwords, name your application (e.g., `Catalyst Tracker`), and click **Generate**.
  • Copy the 16-character code and paste it as the `SMTP_PASS` environment variable on Render.

Render Free-tier Cold Starts

Render's Free Tier spins down web services after 15 minutes of inactivity. When a user visits your app after a period of dormancy, it may take 30–50 seconds to boot up (a "cold start"). This is standard behavior for free hosting and will not occur on paid tiers.

Database Firewall Whitelisting

Ensure that whitelisting on MongoDB Atlas is set to `0.0.0.0/0`. If you restrict database access to your personal home IP, Render's servers will be unable to read or write data, resulting in connection timeouts and 504 Gateway Errors in the client UI.
